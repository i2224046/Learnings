脱・echo！HTMLが主役になるPHPの「代替構文」

PHPをHTMLの中に混ぜて書く際、echoでHTMLタグを出力するのではなく、HTMLの構造はそのままに、プログラムの「つなぎ目」だけをPHPにする書き方が「代替構文」です。

1. 最大の違い：「HTMLを文字列として扱うか、そのまま書くか」

CSV読み込みのコードを例に、「従来の方法」と「代替構文」でどれだけ差が出るか比較してみましょう。ここでは、PHP標準の htmlspecialchars 関数を使って安全に出力する例で説明します。

従来の書き方（echoを多用するスタイル）

PHPのブロックが大きく、HTMLが「文字列」として埋もれてしまいます。

<?php
// ... CSV読み込み処理 ...
foreach ($csv as $row) {
    if ($csv->key() === 0) {
        echo "<tr>";
        echo "<th>" . htmlspecialchars($row[0], ENT_QUOTES, 'UTF-8') . "</th>";
        echo "<th>" . htmlspecialchars($row[1], ENT_QUOTES, 'UTF-8') . "</th>";
        echo "<th>" . htmlspecialchars($row[2], ENT_QUOTES, 'UTF-8') . "</th>";
        echo "<th>他</th>";
        echo "</tr>";
        echo "</thead><tbody>";
        continue;
    }
    echo "<tr>";
    echo "<td>" . htmlspecialchars($row[0], ENT_QUOTES, 'UTF-8') . "</td>";
    echo "<td>" . htmlspecialchars($row[1], ENT_QUOTES, 'UTF-8') . "</td>";
    echo "<td>" . htmlspecialchars($row[2], ENT_QUOTES, 'UTF-8') . "</td>";
    echo "<td><a href='detail.php?id=" . htmlspecialchars($csv->key(), ENT_QUOTES, 'UTF-8') . "'>詳細</a></td>";
    echo "</tr>";
}
?>
⚠️ デメリット

HTMLのタグが "（ダブルクォーテーション）で囲まれているため、エディタでHTMLとして認識されず、打ち間違いに気づきにくいです。また、ドット（.）での連結が多くなり、コードが煩雑になります。

代替構文（HTMLの中にPHPを添えるスタイル）

HTMLが主役になり、PHPは「動き」を制御する役割に徹します。短縮二重引用符 <?= ... ?>（echoの短縮形）を使うとよりスッキリします。

<?php foreach ($csv as $row): ?>
    <?php if ($csv->key() === 0): ?>
        <tr>
            <th><?= htmlspecialchars($row[0], ENT_QUOTES, 'UTF-8'); ?></th>
            <th><?= htmlspecialchars($row[1], ENT_QUOTES, 'UTF-8'); ?></th>
            <th><?= htmlspecialchars($row[2], ENT_QUOTES, 'UTF-8'); ?></th>
            <th>他</th>
        </tr>
        </thead>
        <tbody>
        <?php continue; ?>
    <?php endif; ?>

    <tr>
        <td><?= htmlspecialchars($row[0], ENT_QUOTES, 'UTF-8'); ?></td>
        <td><?= htmlspecialchars($row[1], ENT_QUOTES, 'UTF-8'); ?></td>
        <td><?= htmlspecialchars($row[2], ENT_QUOTES, 'UTF-8'); ?></td>
        <td><a href="detail.php?id=<?= htmlspecialchars($csv->key(), ENT_QUOTES, 'UTF-8'); ?>">詳細</a></td>
    </tr>
<?php endforeach; ?>
✨ メリット

<tr>や<td>がそのまま書けるため、HTMLの階層構造が一目でわかります。

さらに効率化：独自の出力関数を定義する場合

毎回 htmlspecialchars(..., ENT_QUOTES, 'UTF-8') と書くのは大変です。そこで、あらかじめ関数を定義したファイルを require_once しておくと、コードは劇的に美しくなります。この関数を使うと、代替構文の中身はここまでシンプルになります。

<?php foreach ($csv as $row): ?>
    <?php if ($csv->key() === 0): ?>
        <tr>
            <th><?= e($row[0]); ?></th>
            <th><?= e($row[1]); ?></th>
            <th><?= e($row[2]); ?></th>
            <th>他</th>
        </tr>
        </thead>
        <tbody>
        <?php continue; ?>
    <?php endif; ?>

    <tr>
        <td><?= e($row[0]); ?></td>
        <td><?= e($row[1]); ?></td>
        <td><?= e($row[2]); ?></td>
        <td><a href="detail.php?id=<?= e($csv->key()); ?>">詳細</a></td>
    </tr>
<?php endforeach; ?>

「代替構文」と「短い出力関数」を組み合わせることで、HTMLの中にPHPが自然に溶け込み、まるで専用のテンプレートエンジンのような使い心地になります。

2. なぜ「脱・echo」が嬉しいのか？（例え話）

従来のechoスタイルは「電話で家の図面を伝える」

大工さん（ブラウザ）に対して、設計士（PHP）が「次は1行目に左から3センチのところに柱を立てて、その次は……」と、すべて言葉（文字列）で指示を出しているような状態です。これでは全体図がわかりにくいですよね。

代替構文は「透明なシートを重ねる」

HTMLという「完成した図面」がまずあり、その上に「ここだけ数字を変える」「ここを3回繰り返す」という指示が書かれた透明なプログラミング・シート（PHP）を重ねるようなイメージです。土台となる図面が見えているので、間違いが起きにくくなります。

3. この書き方の「3つの快適ポイント」

  1. クォーテーション地獄からの解放
    echo "<a href='...'>" のように、外側をダブル、内側をシングルにするような使い分けに悩む必要がありません。HTMLのルール通りに書けます。

  2. エディタの強力なサポート
    VS Codeなどのエディタが「ここはHTMLだ！」と正しく認識してくれるため、タグの閉じ忘れを教えてくれたり、自動補完（Emmetなど）が使えたりします。

  3. インデントが綺麗に保てる
    HTMLの親子関係（table > tr > td）に合わせて字下げができるため、出力されるソースコードも美しくなります。

4. ルールのおさらい

PHPの動き 従来の記号 { 代替構文の記号 : 閉じ記号
もし〜なら if (...){ if (...): endif;
繰り返す foreach (...){ foreach (...): endforeach;
代替構文は、単なる書き換えではありません。
「PHPの中にHTMLを書く」という考え方から、
「HTMLの中に、必要なときだけ PHPを呼び出す」
という考え方に変えてくれる、非常にクリーンな手法です。
