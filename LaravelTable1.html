Laravel アプリケーション開発ガイド

このドキュメントは、Laravelを用いて「予定表（スケジュール）」を管理する機能を作成する過程をまとめたテクニカルガイドです。

1. 登場する主要な概念

Laravel開発の土台となる重要なキーワードです。

概念 役割・説明
MVC パターン View(見た目)、Controller(司令官)、Model(DB窓口)に役割を分ける設計。
Routing どのURLでどの処理を呼び出すかを決める「地図」。
Migration プログラムで管理するデータベースの「設計図」。
Mass Assignment 保護 予期せぬデータ書き込みを防ぐ。Modelの $fillable で設定。
CSRF 対策 なりすまし送信を防ぐ。フォームに @csrf を書くだけで有効化。
Tailwind CSS class名で直接デザインを指定するモダンなCSSフレームワーク。

2. 使用する Artisan コマンド一覧

コマンド 役割・説明
php artisan make:model Schedule -m モデルと同時にマイグレーションファイルを作成します。
php artisan migrate 設計図を実行し、データベースにテーブルを作成します。
php artisan make:controller ScheduleController 司令官となるコントローラーを作成します。
php artisan migrate:fresh DBを空にして最初からマイグレーションをやり直します。

3. 登場するファイルと役割

ファイルパス 役割・概要
routes/web.php URLと処理の紐付け（ルーティング）。
resources/views/dashboard.blade.php HTMLとデザイン（ビュー）。データの表示やフォーム。
database/migrations/xxxx_create_schedules_table.php DBテーブルの構造定義（マイグレーション）。
app/Models/Schedule.php データの保存・取得の窓口（モデル）。
app/Http/Controllers/ScheduleController.php ビジネスロジックの実行（コントローラー）。

4. やりたいこと別：具体的な作業手順

Step 1

テーブルのデザイン（Tailwind CSS）

枠線を引き、余白を調整します。

<!-- resources/views/dashboard.blade.php -->
<table class="border-collapse">
    <tr>
        <td class="border px-4 py-2">データ</td>
    </tr>
</table>
Step 2

DBの設計図と窓口の作成

1. マイグレーションファイルに項目を追記します。

// database/migrations/...
public function up() {
    Schema::create('schedules', function (Blueprint $table) {
        $table->id();
        $table->string('time');
        $table->text('memo');
        $table->foreignId('user_id');
        $table->timestamps();
    });
}

2. モデルで保存を許可します。

// app/Models/Schedule.php
protected $fillable = ['user_id', 'time', 'memo'];
Step 3

保存処理の実装（Controller）

// app/Http/Controllers/ScheduleController.php
public function store(Request $request) {
    Schedule::create([
        'user_id' => $request->user()->id,
        'time'    => $request->time,
        'memo'    => $request->memo,
    ]);
    return back();
}
Step 4

ルーティングの設定

// routes/web.php
use App\Http\Controllers\ScheduleController;

Route::post('/schedules', [ScheduleController::class, 'store'])->name('schedules.store');

Route::get('/dashboard', function () {
    $schedules = Schedule::where('user_id', auth()->id())->get();
    return view('dashboard', ['schedules' => $schedules]);
})->middleware(['auth']);
Step 5

画面への実装（Blade）

フォームに @csrf を忘れずに。

<form action="/schedules" method="POST">
    @csrf
    <input type="time" name="time">
    <input type="text" name="memo">
    <button type="submit">追加</button>
</form>

@foreach($schedules as $schedule)
    <tr>
        <td>{{ $schedule->time }}</td>
        <td>{{ $schedule->memo }}</td>
    </tr>
@endforeach

5. よくあるエラーと対処法

Undefined variable $schedules

原因: Controllerまたはweb.phpからViewにデータを渡せていない、あるいは変数名が間違っています。

対策: view('dashboard', ['schedules' => $schedules]) の第2引数を確認してください。

419 | PAGE EXPIRED

原因: フォーム送信時にCSRFトークンが送られていません。

対策: <form> の直後に @csrf を追記します。

Table schedules has no column named ...

原因: マイグレーション実行後に項目を追加したが、DBに反映されていません。

対策: php artisan migrate:fresh でDBを再構築します。
