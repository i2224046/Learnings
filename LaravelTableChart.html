Laravel アプリ組織図

各部署をクリックして役割を確認

User / Browser (ユーザー)

アプリを利用する人、およびそのブラウザ（スマホやPC）です。

■ 主な役割

  • 画面（View）のボタンを押したり、フォームに情報を入力する
  • 入力した情報をリクエストとして司令官（Controller）へ送る
  • 返ってきた結果（View）を画面で見る

Controller (司令官)

リクエストを受け取り、各所に命令を出すリーダーです。

■ 主なファイルの場所
app/Http/Controllers/ScheduleController.php など

■ 主に記述する内容

  • ユーザーからの入力データ（Request）の受け取りと確認
  • Modelに対するデータの保存や取得の依頼
  • 処理完了後にどの画面（View）を表示するか、またはどこへ移動するかの決定

■ 具体的なコード例

<?php
namespace App\Http\Controllers;

use App\Models\Schedule;
use Illuminate\Http\Request;

class ScheduleController extends Controller
{
    public function store(Request $request)
    {
        // Viewから送られたデータをModelに渡して保存
        Schedule::create([
            'user_id' => $request->user()->id,
            'time'    => $request->time,
            'memo'    => $request->memo,
        ]);

        // 元の画面に戻るよう指示
        return back();
    }
}

View (現場)

ユーザーが実際に触れる「画面」の担当です。

■ 主なファイルの場所
resources/views/schedule/index.blade.php など

■ 主に記述する内容

  • HTMLタグを用いたページの構造（型紙）やデザイン
  • Controllerから渡されたデータの表示（@foreach などでのループ処理）
  • ユーザーが入力するためのフォーム（@csrf は必須）やボタンの配置

■ 具体的なコード例

<!-- データ送信フォーム -->
<form action="/schedules" method="POST">
    @csrf <!-- セキュリティの合言葉（必須） -->
    <input type="text" name="time" placeholder="時間">
    <textarea name="memo" placeholder="メモ"></textarea>
    <button type="submit">追加</button>
</form>

<!-- Controllerから渡されたデータを一覧表示 -->
<ul>
    @foreach ($schedules as $schedule)
        <li>{{ $schedule->time }} : {{ $schedule->memo }}</li>
    @endforeach
</ul>

Model (データの窓口)

データベース（倉庫）への出入りを厳重に管理します。

■ 主なファイルの場所
app/Models/Schedule.php など

■ 主に記述する内容

  • データベースのテーブルと連携するための基本設定
  • 不正な書き込みを防ぐための「保存許可リスト」（$fillable）
  • データ同士の関係性の定義（例：「この予定は誰のものか」というリレーション）

■ 具体的なコード例

<?php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Schedule extends Model
{
    // 不正な書き込みを防ぐ「保存許可リスト」
    protected $fillable = [
        'user_id',
        'time',
        'memo',
    ];
}

Migration (設計図)

データを格納する「家（テーブル）」の構造を定義します。

■ 主なファイルの場所
database/migrations/日付_create_schedules_table.php

■ 主に記述する内容

  • データベースに新しいテーブルを作成（または削除）する指示
  • テーブル内に持たせる項目（カラム）の名前と種類（文字列のstring、長文のtextなど）

■ 具体的なコード例

<?php
use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up()
    {
        Schema::create('schedules', function (Blueprint $table) {
            $table->id();
            $table->foreignId('user_id'); // 誰のデータか
            $table->string('time');       // 時間（短い文字）
            $table->text('memo');         // メモ（長い文章）
            $table->timestamps();
        });
    }
};

※ Modelと一緒に作成する魔法の呪文：
php artisan make:model Schedule -m

Database (データ保存庫)

ユーザーが入力した情報などが実際に保存される物理的な場所です。

■ 主なファイルの場所（SQLiteの場合）
database/database.sqlite

■ 主な特徴

  • Migration（設計図）で作られた構造（テーブル）に従ってデータを保存する
  • Model（窓口）を通してのみ、データの出し入れが行われる
  • Laravel 11以降は初期状態でファイルベースのSQLiteが使われるため、別途サーバーを用意しなくても動く
